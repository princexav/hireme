## Work Stream 1: UI Polish Bundle

### What Was Built

Four independent UI improvements, all pure presentational — no schema/API changes.

**1. Tailor Sheet Progress Bar (`components/ResumeEditor.tsx`)**
- `progress` state (0–100) drives a shadcn `<Progress>` bar + 4 phase dot indicators
- Deterministic `setTimeout` chain with phase-specific intervals (300ms/430ms/500ms/1000ms per step)
- Holds at 99% until Claude responds, jumps to 100% then clears after 800ms
- Cached resumes (DB hit on mount) show no progress bar — instant load
- Button disabled and relabelled "Tailoring… please wait" while in progress
- `PHASES` array is the single source of truth; `getPhaseIndex`/`getPhaseMs` derive from it

**2. Queue Empty State Card (`app/(dashboard)/queue/page.tsx`)**
- Replaces bare text with: `FileText` icon in `bg-muted rounded-xl p-3` container, heading, body copy, `<Link href="/search">` "Find Jobs" button
- Renders when `queue.length === 0`; existing list renders normally otherwise

**3. QueueCard Salary & Location Row (`components/QueueCard.tsx`)**
- Metadata row below company name: `MapPin` + location, `Banknote` + formatted salary
- `formatSalary()` at module scope, uses `!= null` guards (not falsy — handles `0` correctly)
- `hasMetadata = Boolean(job.location) || Boolean(salaryText)` — explicit boolean
- Row omitted entirely when both are absent

**4. Chat Markdown Rendering (`components/ChatSidebar.tsx`)**
- `react-markdown` applied to assistant bubbles only; user messages remain plain text
- `allowedElements`: `p strong em ul ol li code a`
- `MD_COMPONENTS` extracted to module scope (no inline object on every render)
- All `<a>` tags: `target="_blank" rel="noopener noreferrer"`, `href ?? '#'` guard

## Work Stream 2: JSON Architecture Pivot

### The Problem
Claude was returning a free-form Markdown string as `tailored_text`. This caused:
- LLM hallucinations in formatting
- Character encoding drops (bullets, em-dashes)
- Truncation of older job experience entries

### The Solution
Claude now returns a strict structured JSON object. The PDF renderer maps over it natively.

### Changes

**`lib/claude.ts`**
- New `TailoredResumeJSON` export type:
  ```typescript
  {
    summary: string
    skills: string[]
    certifications: string[]
    experience: Array<{ title, company, location, dates, bullets: string[] }>
    education: Array<{ degree, institution, location, dates }>
  }
  ```
- `tailorResume()` prompt updated: demands `tailored_resume` nested object in the response envelope; `max_tokens` raised from 4096 → 8000
- Anti-truncation constraint added to prompt: *"CRITICAL: You must include EVERY job experience entry provided in the original resume. Do not truncate, summarize, or delete older roles to save space."*
- Return: `JSON.stringify(raw.tailored_resume)` stored into `tailored_text` column (no DB migration needed)

**`app/api/jobs/[id]/tailor/route.ts` — Cache Invalidation**
- Old cache check: `if (existingResume.data?.tailored_text)` → return cached
- New check: wraps `JSON.parse(tailored_text)` in try/catch
  - Valid JSON → return cached (new format)
  - Parse fails → treat as cache miss → re-tailor (graceful invalidation of legacy Markdown)

**`components/ResumePDF.tsx` — Full Rewrite**
- Roboto fonts registered from `public/fonts/Roboto-Regular.ttf` + `public/fonts/Roboto-Bold.ttf`
- Old: line-by-line Markdown splitter + Unicode sanitization regex
- New: `JSON.parse(text)` → structured sections: Summary, Skills, Certifications, Experience (native `•` bullets), Education
- Markdown fallback still present (try/catch) for any stale string that slips through

**`components/ResumeEditor.tsx`**
- `tailored` state now stores pretty-printed JSON: `JSON.stringify(parsed, null, 2)`
- Textarea is `readOnly`, `rows={20}`
- "Copy to Clipboard" copies the formatted JSON string
